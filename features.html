Bootstrap Components & Dynamic Features

This page outlines the Bootstrap components and custom JavaScript features used in my one-page interactive portfolio website for Interactive Media Design and Visualisation CA2.

1. Bootstrap Components

  1. Navbar – Fixed-top responsive navbar with brand, navigation links, and a collapsible menu on smaller screen.
  2. Grid System – The layout uses containers, rows, and columns (.container, .row, .col-*) in the hero, about, projects, experience, skills, hobbies, and contact sections to provide a responsive design when using it on mobile, tablet, and desktop.
  3. Cards – The projects and experience sections use .card components to group related information clearly.
  4. Table – Education details and universities are displayed in a striped, responsive .table.
  5. List Group – Key personal details in the About section are shown using a .list-group.
  6. Progress Bars – Skills are visualised using .progress and .progress-bar components.
  7. Forms – The contact section contains a form built with .form-control and related form classes.
  8. Modal – A Bootstrap .modal is used to show my contact details in a pop-up when the user clicks the Quick Contact buttons.
  9. Buttons & Utilities – Various button styles (.btn classes) and spacing/colour utilities are used throughout the site.

2. Custom JavaScript Features (Non-Bootstrap)

  1. Time-Based Greeting A JavaScript function runs when the page is uploaded and it checks the time on your System. It displays one of the following messages in the hero section: "Good morning", "Good afternoon", or "Good evening", followed by “Welcome to my portfolio”.
  2. Dark Mode Toggle with Persistence A button in the navbar toggles a dark-mode class on the <body>. Extra CSS rules change the background, text colour, card styles, tables, and forms in dark mode (some tables needed different rules because of the bootstratps default white colour for tables). The selected theme is stored in localStorage so that the choice is remembered on future visits.
  3. Dynamic Year in Footer A small script inserts the current year into the footer span with id="year", so the site stays up to date without manual edits.

4. CSS rules inside the project

I used CSS to provide an intereactive and UX design of the website, it includes creating dark mode, resizing the images, making user friendly design, and making the tables to interact when the cursour is beside them.
